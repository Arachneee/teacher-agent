# Design Guidelines

## 컨셉

**귀엽고 심플하다.** 선생님이 수업 전후로 빠르게 사용하는 도구이므로, 불필요한 복잡함 없이 의도가 바로 보여야 한다.

---

## 핵심 원칙

### 1. 의미가 보이는 UI

사용자가 버튼/요소를 보는 순간 용도를 파악할 수 있어야 한다.

- 아이콘보다 텍스트 레이블을 우선한다 (아이콘 단독 사용 지양)
- 레이블은 동사 + 목적어 형태로 명확하게 쓴다. (예: "추가하기", "저장", "복사")
- 상태 변화가 있는 버튼은 상태를 반영한 레이블을 사용한다. (예: "복사됨 ✓")

### 2. 뎁스 최소화

불필요한 확인 단계, 모달, 페이지 이동을 줄인다.

- **삭제처럼 되돌릴 수 없는 동작**만 confirm 또는 추가 확인 단계를 둔다.
- 그 외 수정/저장 등은 인라인으로 처리한다. (페이지 이동 없이 카드 내부에서 편집)
- 단순 입력 후 바로 결과가 보여야 한다. (피드백 생성 → 같은 화면에 결과 표시)

### 3. 귀엽고 둥글둥글한 비주얼

- 모든 카드/버튼/입력창은 `rounded-2xl` 이상의 둥근 모서리를 사용한다.
- 색상은 파스텔 톤을 기본으로 한다. (핑크, 퍼플, 민트 계열)
- 그림자는 가볍게만 사용한다. (`shadow-sm`, `shadow-md` 수준)
- 강조가 필요한 액션 버튼(FAB 등)은 핑크 계열 포인트 색상을 사용한다.

---

## 색상 팔레트

| 용도 | 값 |
|------|----|
| 페이지 배경 | `from-purple-50 via-pink-50 to-orange-50` (그라디언트) |
| 카드 배경 | `white` |
| 주 포인트 | `pink-400` |
| 보조 포인트 | `purple-400` |
| 입력 필드 배경 | `purple-50` |
| 위험 동작 | `rose-400` |
| 일반 텍스트 | `gray-800` |
| 보조 텍스트 | `gray-400` |

## 컴포넌트 규칙

### 카드

- `rounded-3xl`, `shadow-sm`, hover 시 `shadow-md`
- 카드 내부 편집은 인라인 전환 (별도 페이지/모달 없음)
- 삭제 버튼은 카드 내에 두되, 아이콘(🗑️)만 사용해 시각적 무게를 줄인다

### 버튼

- 주요 액션: `bg-pink-400`, `rounded-2xl`, 텍스트 + 이모지
- 보조 액션: `bg-gray-100`, `rounded-2xl`
- 위험 액션: `bg-rose-50 text-rose-400` (배경 연하게, 눈에 덜 띄게)
- FAB(플로팅 버튼): `rounded-full`, `bg-pink-400`, 우하단 고정

### 입력 필드

- `bg-purple-50`, `rounded-2xl`, `outline-none focus:ring-2 focus:ring-purple-300`
- 에러 상태는 `ring-rose-300`으로 표시

### 모달

- 배경: `bg-black/30 backdrop-blur-sm`
- 카드: `rounded-3xl`, `max-w-md`
- 모달 바깥 클릭으로 닫기 지원
- 삭제처럼 위험한 동작에 한해서만 사용

---

## 이모지 사용 지침

이모지는 레이블의 의미를 직관적으로 전달하는 보조 수단으로만 사용한다.

- 페이지 헤더: 서비스 성격을 나타내는 이모지 1개 (예: 🍎)
- 버튼: 텍스트와 함께 사용 (예: "추가하기 ✨", "복사 📋")
- 아이콘 전용 버튼: 의미가 명확한 경우에만 허용 (예: 🗑️ 삭제, ✏️ 수정)

---

## UX 흐름 원칙

```
보기 → 바로 행동 → 결과 확인
```

- 학생 목록: 카드 그리드 → 카드 내에서 바로 편집/삭제
- 피드백 생성: 학생 선택 → 키워드 입력 → 같은 화면에서 AI 결과 확인 → 복사
- 확인(confirm)이 필요한 경우: 삭제뿐
